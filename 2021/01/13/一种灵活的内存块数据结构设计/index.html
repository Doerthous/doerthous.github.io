一种灵活的内存块数据结构设计

一种灵活的内存块数据结构设计

初衷：在dts.net中提供一个数据块结构，用于在网络各层模块中传递数据的同时，尽可能的将动态内存分配推迟到用户层。

完整代码及其实现见dts.net.dblk。

术语

Node：节点
Block：逻辑数据块

1 UDP到IP时的数据变化

假设UDP数据为A，到IP层后，IP层需要在上层的数据的头部新增一个IP头部B，其结构形如链表：B->A。

故此，我们设计一个如下链表进行处理：

typedef struct dblk {
    struct dblk *next;
    uint8_t *data; // 实际数据
    size_t size; // data大小
} dblk_t;

dblk_t *udp;
dblk_t *ip = ip_pack(udp) { // IP模块组包接口
    dblk *ip_header;
    // ...
    ip_header->next = udp;
    return ip_header; 
}

2 链表数据管理

按照1中的分析，假设当前有一个三层网络，每层都会添加头部，那么，加上原始载荷，有：

C->B->A->P。

其中P表示原始载荷，A表示最高层协议，B表示中间层协议，C表示最底层协议。随着层次逐渐降低，链表中的节点逐渐增加。

0层          P
A层          A->P
B层          B->A->P
C层          C->B->A->P

如此，可遍历C、B、A、P节点中的数据完成一个完整协议栈数据的发送或解析操作。

// 将dblk数据拷贝到指定内存
size_t dblk_copy_to(dblk_t *dblk, void *mem, size_t size) {
    size_t do_size;
    size_t done_size = 0;
    while (dblk) {
        do_size = size < dblk->size ? size : dblk->size;
        memcpy(mem, dblk->data, do_size);
        done_size += do_size;
        mem += do_size;
        if (done_size == size) {
            break;
        }
        dblk = dblk->next;
    }
    return done_size;
}

send(dblk) {
    uint8_t buff[SOME_SIZE];
    size_t size = dblk_copy_to(dblk, buff, SOME_SIZE);
    low_level_send(buff, size);
}

3 [2]中存在的问题

考虑到底层可能由于硬件原因，无法及时发送数据，导致多个完整协议栈数据被缓存。为管理这些数据，可添加用于表示后续节点是否属于当前逻辑数据块的more标志，区分不同的逻辑数据块，具体解释如下图：

C1 -> B1 -> A1 -> P1 -> C2 -> B2 -> A2 -> P2 -> ...
m:1   m:1   m:1   m:0   m:1   m:1   m:1   m:0   ...

上述链表虽有8个节点（C1 ~ P2），但是只有两个逻辑数据块，其中（C1 ~ P1）是一个，（C2 ~ P2）是另一个（m：more）。在处理一个节点的数据时，需要判断more标志，如果more标志为1则表示后续节点仍属于当前逻辑数据块，否者则表示当前节点为当前逻辑数据块的最后一个节点。类似IP报文中的More Fragment标志。

如此，在处理时可按照逻辑数据块进行区分：

size_t dblk_copy_to(dblk_t *dblk, void *mem, size_t size) {
    size_t do_size;
    size_t done_size = 0;
    while (dblk) {
        do_size = size < dblk->size ? size : dblk->size;
        memcpy(mem, dblk->data, do_size);
        done_size += do_size;
        mem += do_size;
        if (done_size == size || !dblk->more) { // 按逻辑数据块进行处理
            break;
        }
        dblk = dblk->next;
    }
    return done_size;
}

除上述方法外，也可引入另一个指针来连接不同的逻辑数据块。

C1 -> B1 -> A1 -> P1
 |
 v
C2 -> B2 -> A2 -> P2

4 分片及内存对齐问题

假设某个双层应用层协议需通过UDP传输，最终形成如下内存：

U  ->  L1    ->    L2
\       \           \
+-----+   +------+   +------+
|  U  |   |  L1  |   |  L2  |
+-----+   +------+   +------+

在逻辑上，我们可以将上述内存想象成一大块连续分配的虚拟内存：

+-----+------+------+
|  U  |  L1  |  L2  |
+-----+------+------+

到IP层时，由于逻辑数据块长度太大，IP模块需要进行分片，为了最大化节省空间，我们可让IP层节点指向上述虚拟内存。例如，假设经计算上述内存需被分为4个IP分片报文：

F1   F2   F3   F4
|    |    |    |
+-----+------+------+
|  U  |  L1  |  L2  |
+-----+------+------+

IP1 -> F1
IP2 -> F2
IP3 -> F3
IP4 -> F4

每个IPx节点的内存中包含了IP Header的数据，而Fx节点包含了传给IP层的上层数据分片。按照上述虚拟的连续内存思路，我们可以实现利用原数据而无需重新拷贝而得到分片的操作。然而，由于IP层在分片时对载荷有对齐要求（8B对齐），如果U、L1、L2内存本身没有满足8B对齐，那么就会出现上图中类似F2与F3的情况：一个dblk节点的所代表的内存块在实际物理内存上并不连续！

为此，我们引入一个vsize和类型为dblk_t *的vmem指针，用vsize表示当前节点虚拟内存的大小，当当前节点的data的大小无法满足vsize时，我们继续从vmem中取内存，直到其大小为vsize为止。

在保持原有链表不动的情况下，假设U、L1、L2的data指针分别为u、l1、l2，大小分别为us、l1s、l2s，如下所示，

U  ->  L1  ->  L2
d:u    d:l1    d:l2
s:us   s:l1s   s:l2s

再设分片后，每个分片报文大小为x，则，F1 ~ F4的data指针及其内容如下：

F1   ->   F2   ->   F3  ->                 F4
d:u       d:u+x     d:l1+(x-(us-x))        d:l2+(x-(l1s-(x-(us-x))))
s:us      s:us-x    s:l1s-(x-(us-x))       s:l2s-(x-(l1s-(x-(us-x))))
vs:x      vs:x      vs:x                   vs:l2s-(x-(l1s-(x-(us-x))))
vm:NULL   vm:L1     vm:L2                  vm:NULL

推导：

  1. F1节点，d指向u；s为d的大小us；vs=x；由于us>x，故F1无需再取内存，故vm为NULL。

  2. F2节点，由于F1没能占完u，故F2的d指向u加上F1占用u后剩下的内存，F1占了x，所以F2的d指向u+x；s为d的大小(us-x)；vs=x；从图中可看出，仅仅u无法满足F2的vs大小，F2后续还占用了l1，故vm=L1。

  3. F3，F4思路类似。

需要注意的是F4作为尾节点，可能没有vmem使用，故其vs可能不一定等于x。此外一个vmem-节点可能占用原来链表的多个节点内存，如下图，Fa占用了三块原节点内存。

Fa             Fb
|              |
+-----+------+------+
|  U  |  L1  |  L2  |
+-----+------+------+

在此过程中，size字段的含义不变，仍旧表示data的大小。只是当未启用vmem时，节点的内存大小正好等于data的大小。而vmem启用时，节点的内存大小未必等于data的大小。按照上述思路，可以设计一个如下接口，实现将dblk内存分割成按任意字节对齐的片段。需要注意，此处的dblk参数必须是未启用vmem模式的节点。

dblk_t *dblk_fragments(dblk_t *dblk, size_t frag_size);

至此，当上层数据超长需要分片时，仅需调用dblk_fragments进行处理即可。可以引入一个将vmem节点还原回正常节点的操作，如此便可无限制地使用分片接口。

5 接口

Node结构及接口

typedef struct dblk
{
    struct dblk *next;
    uint8_t *data;
    uint16_t size;
    uint16_t data_malloc:1; // 指示data字段是否来自malloc
    uint16_t node_malloc:1; // 指示此Node是否来自malloc。

    /**
     * @brief 指示后续Node与当前Node同属一个Block。
     *
     *  假设有如下dblk链表:
     *   Nodes:
     *   n1    ->    n2    ->   n3    ->    n4    ->    n5    ->  n6
     *   more:1      more:1     more:0      more:1      more:1    more:0
     *   -----------------------------      ----------------------------
     *   Block1                             Block2
     */
    uint16_t more:1;

    /**
     * @brief vmem模式.
     * 
     *  当vsize != 0的Node属于vmem-Node，反之为rmem-Node。vsize表示当前vmem-Node所
     *  带内存大小，而size表示data的大小。
     */
    uint16_t vsize;
    struct dblk *vmem;
} dblk_t;

6 测试

int main()
{
    dblk_t *b1 = dblk_node_new_with_buff(12);
    dblk_t *b2 = dblk_node_new_with_buff(12);
    dblk_t *b3 = dblk_node_new_with_buff(12);
    dblk_node_concat(b1, b2);
    dblk_node_concat(b2, b3);

    dblk_t *it = b1;
    while (it) {
        for (int i = 0; i < it->size; ++i) {
            printf("%02X ", it->data[i]);
        }
        it = dblk_node_next(it);
    }
    printf("\n");

    dblk_copy_from(b1, "123456789098765432101234567890123456", 36);
    char buff[36];
    {
        int c = (int)dblk_copy_to(b1, buff, 36);
        for (int i = 0; i < c; ++i) {
            printf("%02X ", buff[i]);
        }
        printf("\n");
    }

    dblk_t *fragments = dblk_fragment(b1, 7);
    it = fragments;
    while (it) {
        memset(buff, 0, 36);
        int c = (int)dblk_copy_to(it, buff, 36);
        printf("%sn: ", dblk_node_is_vmem_node(it)?"v":" ");
        for (int i = 0; i < c; ++i) {
             printf("%02X ", buff[i]);
        }
        printf("\n");
        it = dblk_next(it);
    }

    if (dblk_list_to_rmem_list(fragments)) {
        it = fragments;
        while (it) {
            memset(buff, 0, 36);
            int c = (int)dblk_copy_to(it, buff, 36);
            printf("%sn: ", dblk_node_is_vmem_node(it)?"v":" ");
            for (int i = 0; i < c; ++i) {
                printf("%02X ", buff[i]);
            }
            printf("\n");
            it = dblk_next(it);
        }
    }
    dblk_list_delete(fragments);


    fragments = dblk_fragment(b1, 5);
    it = fragments;
    while (it) {
        memset(buff, 0, 36);
        int c = (int)dblk_copy_to(it, buff, 36);
        printf("%sn: ", dblk_node_is_vmem_node(it)?"v":" ");
        for (int i = 0; i < c; ++i) {
             printf("%02X ", buff[i]);
        }
        printf("\n");
        it = dblk_next(it);
    }
    dblk_list_delete(fragments);

    dblk_list_delete(b1);

    return 0;
}
